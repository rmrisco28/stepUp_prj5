# Step Up 📚

![Java](https://img.shields.io/badge/Java-17+-brightgreen)
![Spring Boot](https://img.shields.io/badge/Spring%20Boot-3.x-brightgreen)
![React](https://img.shields.io/badge/React-18+-blue)

**클라우드 기반 학생 역량 통합 관리 플랫폼**

대학생들의 비교과 활동과 핵심역량을 체계적으로 관리하고, 역량 진단을 통해 자기계발을 지원하는 통합 관리 시스템입니다.

## 🌟 프로젝트 배경

대학생들의 자기계발과 역량 강화를 위해 다양한 비교과 활동 참여가 증가하고 있으며, 이를 체계적으로 관리할 수 있는 통합 관리 체계의 필요성이 대두되었습니다. Step Up은 이러한 요구를 충족시키기 위해 기획된 클라우드 기반 역량 관리 플랫폼입니다.

## 🎯 주요 기능

### 📋 비교과 프로그램 관리

- **프로그램 등록 및 관리**: 관리자가 비교과 프로그램 생성, 수정, 삭제
- **프로그램 신청**: 학생이 원하는 프로그램 신청 및 취소
- **승인 및 참여 관리**: 신청 승인 처리 및 참여 현황 관리
- **이수 내역 조회**: 학생별 프로그램 이수 내역 확인
- **이미지 관리**: AWS S3를 활용한 프로그램 이미지 업로드 및 관리

### 🎓 핵심역량 관리

- **핵심역량 등록**: 학교별 핵심역량 및 하위역량 정의
- **역량 진단 검사**: 학생들의 현재 역량 수준 진단
- **진단 결과 분석**: 역량별 검사 결과 확인 및 피드백 제공
- **역량별 프로그램 연계**: 부족한 역량 개발을 위한 프로그램 추천

### 👥 회원 관리

- **학번/사번 자동 생성**: Spring Batch를 활용한 대량 회원 데이터 처리
- **로그인/로그아웃**: 세션 기반 인증 시스템
- **권한 관리**: 학생, 교직원 역할별 권한 관리
- **비밀번호 변경**: 개인정보 보호를 위한 비밀번호 변경 기능

### 📢 게시판 관리

- **공지사항**: 중요 알림 및 공지사항 게시
- **FAQ**: 자주 묻는 질문 관리 및 조회

## 🛠️ 기술 스택

### Backend

- **Framework**: Spring Boot 3.x
- **Security**: Spring Session (세션 기반 인증)
- **Database**: MySQL (JPA/Hibernate)
- **Batch Processing**: Spring Batch (대량 데이터 처리)
- **File Storage**: AWS S3
- **Build Tool**: Maven

### Frontend

- **Framework**: React 18+
- **Styling**: CSS3, Bootstrap
- **HTTP Client**: Axios
- **Routing**: React Router

### Infrastructure

- **Cloud Platform**: AWS
  - **EC2**: 애플리케이션 서버
  - **RDS**: MySQL 데이터베이스
  - **S3**: 이미지 및 파일 스토리지
- **CI/CD**: 클라우드 기반 배포

```
프로그램 생성 → 프로그램 신청 및 승인 → 참여 및 이수 → 이수 내역 관리
```

#### 2️⃣ 핵심역량 진단 프로세스


## 🚀 시작하기

### 필요 조건

- Java 17+
- Node.js 18+
- MySQL 8.0+
- Maven 3.6+
- AWS 계정 (S3, RDS, EC2)

### 설치 및 실행

1. **저장소 클론**
   ```bash
   git clone https://github.com/rmrisco28/stepUp_prj5.git
   cd stepUp_prj5
   ```

2. **Backend 설정**
   ```bash
   cd backend
   
   # application.properties 설정
   # - 데이터베이스 연결 정보
   # - AWS S3 설정
   # - 세션 관리 설정
   
   # Spring Batch용 CSV 파일 준비
   # src/main/resources 폴더에 학생/교직원 정보 CSV 업로드
   
   # 실행
   ./mvnw spring-boot:run
   ```

3. **Frontend 설정**
   ```bash
   cd frontend
   npm install
   npm run dev
   ```

4. **접속**
   - Frontend: http://localhost:3000
   - Backend API: http://localhost:8080

## 🔄 Spring Batch 사용법

### 학번/사번 생성 배치

1. `src/main/resources` 폴더에 CSV 파일 업로드
2. 배치 실행 선택 (학생 또는 교직원)
3. 자동으로 학번/사번 생성 및 DB 저장
4. 초기 비밀번호 암호화 처리
5. 배치 메타데이터 테이블에 결과 저장

### 배치 처리 흐름

```
CSV 업로드 → 데이터 가공 → DB 저장(1단계) → 
학번/비밀번호 암호화(2단계) → SEQ 매핑(3단계) → 완료
```

## 🐛 트러블 슈팅

### 1. 역량 진단 검사 시 이동 및 저장 버튼 문제

**문제**: 페이지네이션 추가 후 페이지 이동 시 답안이 DB에 저장되지 않는 문제 발생

**해결**:
- 페이지 이동 시 현재 답안 자동 저장 로직 추가
- 이전 페이지 이동 시 DB 저장 값 불러오기 구현
- 답안 수정 시 기존 값 업데이트
- 중복 저장 방지 및 빈 값 이동 제한으로 안정성 확보

### 2. 비교과 프로그램 신청 조회 문제

**문제**: Member, Application, Program 간 복잡한 연관관계로 인한 매핑 오류

**해결**:
- 필요한 데이터만 포함한 전용 DTO 생성
- Program → Application → Complete 순서로 단계별 조회
- Stream API 활용하여 데이터 변환 및 DTO 생성을 함수형으로 처리

### 3. 이미지 수정 시 삭제 오류

**문제**: 프로그램 수정 시 본문 이미지가 삭제되지 않는 문제

**해결**:
- fileName이 전체 URL로 들어오는 것을 확인
- `substring`과 `lastIndexOf`를 통해 순수 파일명 추출
- S3 삭제 로직 수정으로 정상 동작 확보
